# schreier_sims.cpp — API Reference

Implements the Monte Carlo (randomized) Schreier-Sims algorithm to compute a
**Base and Strong Generating Set (BSGS)** for a permutation group.

---

## Types

```cpp
using Perm = std::vector<int>;
```

A permutation on `{0, …, n−1}` stored as a zero-indexed array: `p[i]` is the
image of point `i`.

---

## Core function

```cpp
BSGS randomized_schreier_sims(
    int n,
    const std::vector<Perm>& generators,
    int      confidence = 40,
    uint32_t seed       = 42);
```

Computes a BSGS for the group generated by `generators` acting on `{0, …, n−1}`.

| Parameter | Description |
|-----------|-------------|
| `n` | Degree of the group (number of points). |
| `generators` | Generating permutations, each of length `n`. |
| `confidence` | Stop after this many consecutive trivial sifts. Error probability ≤ 2^(−confidence). Default 40 ≈ 10^(−12). |
| `seed` | RNG seed for reproducibility. |

**Returns** a `BSGS` object representing the stabilizer chain.

---

## BSGS struct

```cpp
struct BSGS {
    int n;
    std::vector<Level> chain;
};
```

### Methods

```cpp
long long BSGS::order() const;
```
Returns the order (size) of the group: product of all orbit sizes in the chain.

```cpp
std::vector<int> BSGS::base() const;
```
Returns the base sequence `[β₀, β₁, …]`.

```cpp
std::vector<std::vector<Perm>> BSGS::sgs_per_level() const;
```
Returns the Strong Generating Set grouped by stabilizer level.

```cpp
Perm BSGS::sift(Perm g) const;
```
Strips `g` through the stabilizer chain. Returns the residue permutation.
The residue is the identity iff `g ∈ G` (assuming the chain is complete).

```cpp
Perm BSGS::canonicalize(Perm s) const;
```
Returns the canonical (lexicographically minimal) left-coset representative of
`s` under the group action. Two permutations are in the same left coset iff
their canonical forms are equal:
```
canonicalize(compose(g, s)) == canonicalize(s)  for every g in G
```

```cpp
bool BSGS::augment(Perm g);
```
Inserts `g` into the chain, extending orbits and adding a new level if needed.
Returns `true` if the chain was modified. Mainly used internally during
construction; can also be called after the fact to extend a partial chain.

---

## Utility functions

```cpp
bool membership_test(const BSGS& bsgs, const Perm& g);
```
Returns `true` iff `g` belongs to the group represented by `bsgs`.
Equivalent to `is_identity(bsgs.sift(g))`.

```cpp
bool subgroup_test(const BSGS& H, const BSGS& G);
```
Returns `true` iff `H ≤ G`. Checks that every SGS generator of `H` is a
member of `G`; since the SGS generates `H`, this is sufficient.

## WebAssembly (Emscripten) API

When compiled with Emscripten (`-s BIND`), the module exposes
`SchreierSimsRunner` and a helper vector type.

### Registered types

| JS name | C++ type |
|---------|----------|
| `VectorInt` | `std::vector<int>` |
| `SchreierSimsRunner` | `SchreierSimsRunner` |

### SchreierSimsRunner

```js
const runner = new Module.SchreierSimsRunner();
```

#### `reset(n: number): void`
Set the degree to `n` and clear all previously added generators.

#### `addGenerator(g: VectorInt): void`
Add one generator permutation. `g` must be a `Module.VectorInt` of length `n`.

```js
const g = new Module.VectorInt();
[1, 2, 0, 3].forEach(x => g.push_back(x));
runner.addGenerator(g);
g.delete();
```

#### `run(confidence: number): string`
Run the Monte Carlo Schreier-Sims algorithm and return a JSON string:

```json
{
  "base":       [0, 2],
  "orbitSizes": [6, 4],
  "order":      24
}
```

### Full JS example

```js
runner.reset(4);

const gens = [[1,2,3,0], [1,0,2,3]];   // S₄: (0123) and (01)
for (const g of gens) {
    const v = new Module.VectorInt();
    g.forEach(x => v.push_back(x));
    runner.addGenerator(v);
    v.delete();
}

const result = JSON.parse(runner.run(40));
console.log(result.order);   // 24
```

## CLI (native build)

```
g++ -O2 -std=c++17 -o schreier_sims schreier_sims.cpp
./schreier_sims             # runs built-in test cases
./schreier_sims --stdin     # reads from stdin: n t \n g₁ \n g₂ …
```

stdin format:
```
n t
p[0] p[1] … p[n-1]    ← generator 1
…                      ← generator t
```
